# サマリー(目次)とページ

### サマリー(目次)

HonKitは`SUMMARY.md` ファイルで、ブック上のチャプター(章・節・項・目)構成の定義(章立て)をします。`SUMMARY.md` ファイルは、ブックの目次を生成するために使用されます。

`SUMMARY.md`のフォーマットは、単なるリンクの箇条書きです。リンクタイトルはそのチャプターのタイトル、リンク先はそのチャプターのファイルのパスにします。

親チャプターの配下に、ネスト(入れ子構造)にしたリンクを追加すると、子チャプターが作成されます。

##### 簡単な例

```markdown
# サマリー

* [パート1](part1/README.md)
    * [書くことはいいことだ](part1/writing.md)
    * [HonKitっていいな](part1/honkit.md)
* [パート2](part2/README.md)
    * [フィードバックが大好き](part2/feedback_please.md)
    * [執筆者のための良きツール](part2/better_tools.md)
```

この例では、親チャプター別の専用ページ(`part〇/README.md`)を持ち、さらに子チャプターを持つ構成になっています。

##### アンカー

サマリー(目次)の各チャプターは、アンカー(ページ内リンク)を使ってファイル中の特定箇所の指定ができます。

```markdown
# サマリー

* [パート1](part1/README.md)
    * [書くことはいいことだ](part1/README.md#writing)
    * [HonKitっていいな](part1/README.md#honkit)
* [パート2](part2/README.md)
    * [フィードバックが大好き](part2/README.md#feedback)
    * [執筆者のための良きツール](part2/README.md#tools)
```


##### パート分け

目次は、見出しや横線を用いることで、区切られたパート(部・編)に分割することができます。

```markdown
# サマリー

### パートI

* [書くことはいいことだ](part1/writing.md)
* [HonKitっていいな](part1/honkit.md)

### パートII

* [フィードバックが大好き](part2/feedback_please.md)
* [執筆者のための良きツール](part2/better_tools.md)

----

* [タイトルのないパート](part3/title.md)
```

パートは単なるチャプターのグループ分けをしているだけで、個別の専用ページは持ちませんが、ブックのテーマに応じて、ナビゲーション(メニュー)の表示に反映されます。

### ページ

#### Markdownの構文

HonKitのファイルのほとんどは、デフォルトでMarkdownの構文を使用します。HonKitはそこからあなたのページの構造を判断します。使用される構文は[GitHub Flavored Markdown 構文](https://guides.github.com/features/mastering-markdown/)に似ています。また、[AsciiDoc 構文](asciidoc.md)を使うこともできます。

#####チャプターファイルの例

``` markdown
# チャプターのタイトル

これは素晴らしい導入です。

## セクション1

Markdownはあなたの**ブックの構造**の_ほとんど_を決定します

## セクション2

...

```

#### フロントマター

ページにはオプションでフロントマターを含めることができます。これはページの説明を定義するために使用することができます。前書きはファイルの中で最初のものでなければならず、三重のハイフンで囲まれた有効なYAMLの形式を取らなければなりません。以下は基本的な例です。

```yaml
---
description: これはページについての短い説明です。
---

# 私のページのコンテンツ
...
```

フロントマターでは独自の変数を定義することができ、それらは[ページ変数](templating/variables.md)に追加されますので、テンプレートで使用することができます。この場合、`{{page.description}}`とこのページ中に記載すれば、上記の変数の値を出力できます。
